# [ADR-007] kanon-core と kanon-twin の分離可否の判断

## ステータス
**承認済 (Accepted)** - 2026-06-20T21:24:39+09:00

---

## 1. 背景 (Context)

長期記憶を持つ自律エージェント基盤を整備するにあたり、システム管理やツール実行を担う**「エージェント・コア (Kanon Core)」**と、個人のキャリアやヘルスログなどの機密性の高い記憶情報を保有しミラーリングを行う**「デジタルツイン (Kanon Twin)」**の2つの役割が定義されました。

これらは将来的には別々のインフラ（例：パブリッククラウド上の Core エージェントと、ローカルの安全な自宅サーバー上の Twin 記憶DB）で物理的に分離して稼働することが想定されます。このため、開発の初期段階（Phase 0）である現時点からリポジトリを `kanon-core` と `kanon-twin` に分割すべきか、あるいは同一リポジトリ内で論理的な境界を定義するに留めるべきかを評価します。

---

## 2. 評価観点 (Evaluation Metrics)

### ① Complexity (複雑性)
* **分離した場合**:
  * 共通ライブラリやユーティリティ（ログ処理、LLM Providerの抽象化、検証スクリプト等）を双方で共有するための仕組み（パッケージング、サブモジュール、あるいは二重定義）が必要になり、コードの依存関係が複雑化します。
* **分離しない場合**:
  * 同一リポジトリ内で Python モジュール間のインポートのみで完結するため、極めてシンプルです。

### ② Future Growth (将来の拡張性)
* **分離した場合**:
  * 最初から物理的分離を見越した独立設計（API経由での通信など）を強制されるため、将来的なスケールアウトや、別のインフラ環境への配備がスムーズに行えます。
* **分離しない場合**:
  * モジュール間が密結合になりやすく、将来的に別リポジトリや別コンテナへ引き剥がす際の移行コストが発生します。

### ③ Maintenance Cost (メンテナンスコスト)
* **分離した場合**:
  * 2つのリポジトリに対する二重の CI/CD 設定、Docker イメージビルド、およびGitコミット履歴の分散管理が発生し、オーバーヘッドが肥大化します。
* **分離しない場合**:
  * 1つのリポジトリで一元管理できるため、インフラのデプロイやテストが容易であり、現在の小規模な開発段階においてはコストを最小化できます。

### ④ Repository Governance (リポジトリガバナンス)
* **分離した場合**:
  * 各リポジトリで `AGENTS.md` (憲法) や `docs/` を個別に同期・同期維持する必要があり、ガバナンスに乖離が生じるリスクがあります。
* **分離しない場合**:
  * 単一の `AGENTS.md` のもとで、エージェントと人間が共通のルール・設計決定（ADR）を完全に共有でき、Docs as Code のガバナンスが一貫します。

---

## 3. 決定 (Decision)

### 結論: **Split Later (将来的な分離)** を推奨・採用

現時点ではリポジトリを物理的に分離せず、**`kanon` 単一リポジトリ内での論理的境界の分割**に留めます。

* **運用の詳細**:
  * `ai-agent/workspace/src/` 配下で、エージェントのインフラ/コア機能と、デジタルツイン用インターフェースのフォルダ・クラス定義を論理的に分離した疎結合設計を維持します。
  * 将来的に **Phase 3 (Digital Twin)** が本格化し、個人情報保護の観点から別ホストでの稼働、または別コンテナ化による権限制御が必要になった段階で、正式にリポジトリを `kanon-core` と `kanon-twin` に分割（Split）します。

---

## 4. 影響 (Consequences)

* **メリット**:
  * Phase 0 および Phase 1 (生存基盤構築) において、開発・運用のシンプルさと高速性を最大化できます。
  * `AGENTS.md`（エージェント憲章）の一元化により、ガバナンスが容易になります。
* **デメリット**:
  * 将来の分離フェーズ移行時に、疎結合設計が守られていないと、依存コードの分離・リファクタリングコストが発生します（これを防ぐため、コード設計時は常に `Core` と `Twin` の疎結合を意識する必要があります）。
