## 1. 🎯 現在の目的 (Current Goal)

当初は「自己改変Agent」の実験として開始したが、本来のビジョンに立ち返り、以下の目的を最優先とする「長期記憶を持つ Agent OS」として再定義する。
1. **長期記憶基盤** (Markdown + Git 管理)
2. **デジタルツイン** (人間をミラーリング・代行する相棒)
3. **半自律リサーチAgent** (自律調査・提案・分析)
4. **長期運用可能なAgent基盤** (Survival First)

※ 自己改変は目的ではなく、これらを支える補助機能（Supporting Capability）に位置づける。

---

## 2. 🚦 現在の状況 (Current State)

### ① 技術状況
* **物理ホスト**: GPD WIN 3 (Ubuntu Desktop)
* **実行環境**: Dockerによるコンテナ運用
* **API鍵類**: Gemini API Key, Discord Bot Token 設定済み
* **接続確認**: Discord DM接続確認済み、Gemini動作確認済み（503リトライは未実装）

### ② Repository設計・境界の現状
* **非特権エージェント**: エージェントコンテナは非特権で実行。
* **特権コントローラー**: `backup`, `rollback`, `restart`, `healthcheck` などの特権操作を担当。
* **Secretsの隔離**: エージェントコンテナから `secrets/` は直接参照不可。
* **Runtime**: 生成物（`logs/`, `state/`, `backups/`, `candidates/`）をマウント領域で管理。
* **Docs as Code**: `docs/` ディレクトリを構造化（`architecture/`, `decisions/`, `lessons/`, `roadmap/`, `worklog/`, `templates/`）し、変更時にはドキュメントもAIが自律更新。

### ③ 憲法・ルール (AGENTS.md 方針)
* **優先順位**: 1. Survival > 2. Observability > 3. Recovery > 4. Improvement > 5. Performance
* **絶対禁止事項**: `secrets` 変更、`backup` 削除、検証なし更新、ドキュメントのサマリーなし破壊的更新。
