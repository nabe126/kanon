## 4. 🚀 次に推奨されるアクション (Next Recommended Actions)

人間（ユーザー）または次のエージェントは、以下の手順で実機（GPD WIN 3 / Ubuntu / Docker）検証を実施してください。

### 【ステップ1: 実機でのコンテナ起動】
1. リポジトリを実機に clone/pull。
2. `secrets/.env` に本物の `DISCORD_BOT_TOKEN`, `GEMINI_API_KEY` を配置。
3. `KANON_TEST_TRIGGER=true` を環境変数（または `.env` 内）に追加設定。
4. `docker-compose up --build -d` でコンテナを起動。

### 【ステップ2: healthz 疎通とメトリクス取得確認 (L2/L3)】
* ホスト（Ubuntu）側から `curl http://localhost:5000/healthz` を叩き、実機の CPU/メモリ/ディスクメトリクスが正しくパースされ、`status: healthy` になっていることを確認。

### 【ステップ3: モニター起動と自動ロールバック実証 (L2/L3)】
1. ホスト側でロールバックモニターを手動起動する。
   ```bash
   python controller/monitor.py
   ```
   （起動時に「`Agent is healthy at startup. Registering initial LKG snapshot...`」と表示され、LKG初期スナップショットが作られることを確認）
2. ホスト側から、テストハーネスを叩いて擬似障害（500 Unhealthy）を意図的に引き起こす。
   ```bash
   curl http://localhost:5000/healthz/fail
   ```
3. `monitor.py` が連続で 3 回ヘルスチェック失敗を検知し、`Initiating automatic rollback procedure...` を実行することを確認。
4. `shutil.rmtree` および `copytree` により、LKG スナップショットから `src/` が完全復旧され、`docker restart ai_agent_core` が実行されることを確認。
5. 60秒のクールダウン後、コンテナが healthy（元の正常な状態）に自動復帰することを確認できれば、**Phase 1 Exit Criteria 達成** となります。
