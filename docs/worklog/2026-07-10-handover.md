## 3. 🚀 次回（以降）のタスクと手順 (Next Tasks & Steps)

> [!IMPORTANT]
> 次回は、決定された **「完全自動PR起票フロー」** の実現に向け、以下のステップに沿って開発および設定を行います。

### Step 1: 【人間作業】Bot用の GitHub アカウントとトークンの用意
エージェントが自律的に PR を起票するため、人間側でアカウントとトークンを用意し、実機に設定します。

1. 手元のブラウザ等から、Bot専用の GitHub アカウント（例: `kanon-arahabaki-bot` 等）を新規に1つ作成します。
2. 作成したBotアカウントで **Personal Access Token (PAT)** を発行します（必要な権限: `repo`）。
3. 実機ホストの `ai-agent/secrets/.env` ファイルに、以下のようにトークンを設定（追記）します：
   ```bash
   GITHUB_TOKEN=ghp_xxxxxxxxxxxx
   ```

### Step 2: 【エージェント作業】REST API 経由でのPR自動作成モジュールの実装
次回起動したエージェントは、`.env` に追記された `GITHUB_TOKEN` を使用して、GitHub API を直接叩き、PRを自動作成する機能を実装します。

* **実装方針**:
  * コンテナ内に重い外部 CLI ツールをインストールするのを避けるため、Python の `urllib.request` または `requests` を使用して、GitHub REST API (`POST /repos/{owner}/{repo}/pulls`) を直接叩いて PR を起票する軽量なPR作成機能を [agent_tools.py](file:///workspace/ai-agent/workspace/src/utils/agent_tools.py) などに実装します。
  * エージェントが開発用指示を受け、コード修正とテスト検証に成功した際に、自動で作業用ブランチ（例: `fix/task-xxxx`）を作成してコミット・プッシュし、上記 API を叩いてプルリクエストを作成する一連の Git 連携フローを実装・統合します。
