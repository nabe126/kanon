## 4. 🚀 次に推奨されるアクション (Next Recommended Actions)

> [!IMPORTANT]
> 次回の作業開始時、または人間が実機を触る際に、以下の **3つのステップ** を上から順に実行して、最新の「自律開発エージェント」を立ち上げてください。

### Step 1: 実機のファイル権限・競合をクリアして同期する
実機（GPD）のターミナル（`~/kanon` ディレクトリ）で以下を実行し、Git pullのエラー原因を排除して最新コードを同期します。

```bash
# ① 起動中の古いコンテナを停止します
make down

# ② コンテナ内のroot権限で上書きされてしまった所有権をホストユーザーに戻します
sudo chown -R $(whoami):$(whoami) .

# ③ 実機上で自動生成された一時データや競合する未追跡ファイルを強制クリーンアップします
git reset --hard HEAD
git clean -fd

# ④ GitHub から最新コード（自律エージェント版）を同期します
git pull
```

### Step 2: コンテナの再ビルド＆起動
コードが正常に同期できたら、コンテナを起動します。

```bash
# 再ビルドしてバックグラウンドで起動
make up
```
* `make status` を実行し、`ai_agent_core` と `ai_agent_monitor` が共に `Up` になっていることを確認してください。

### Step 3: Discord上での自律エージェントの動作テスト
Bot（`@GPT-agent`）に対し、Discord上で直接以下の指示をチャットして挙動をテストします。

1. **テスト①（リポジトリの探索と読込）**
   * 👤 **ユーザー**: `@GPT-agent README.mdの概要を3行で教えて`
   * 🤖 **エージェント**: 自動的に `read_file` ツールを実行し、中身を読み取って回答を返せば成功です。
2. **テスト②（コマンドの実行とボタン連携）**
   * 👤 **ユーザー**: `@GPT-agent make statusを実行して状態を教えて`
   * 🤖 **エージェント**: ツール `request_command_execution` を呼んで計画を起票し、チャットに **`[✅ 承認 (YES)]` `[🛑 却下 (NO)]`** のボタンを添付して返します。
   * 👤 **ユーザー**: **`[✅ 承認 (YES)]`** をクリック ➔ コマンドが実行され、結果がチャットに返ってくれば大成功です！
