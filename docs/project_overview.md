## 🔍 1. プロジェクト概要 (What is Kanon?)

* **開発コード**: `arahabaki`
* **プロダクト名**: `Kanon`
* **コアコンセプト**: 
  Kanonは、単に指示されたコードを書くだけのAIではありません。**「人間と協調しながら、自ら課題を発見し、プロジェクトを主体的に育てる自律型開発パートナー（AI）」** を目指しています。

### 🧠 基本哲学（GitHub ＝ AIの外部脳）
Kanonは、GitHubの各種機能を「ソースコードの置き場所」ではなく「自身の長期記憶・認知インターフェース」として活用します。
* **Issue** ➔ **未来の仕事（タスク・課題）**
* **Pull Request (PR)** ➔ **実施内容（行動履歴）**
* **Review** ➔ **人間からの教育・フィードバック**
* **Projects** ➔ **現在の脳（タスクの優先順位と状態）**

## 🏛️ 2. 現在のアーキテクチャと動作環境

実機（GPDホスト）上でDockerコンテナを起動し、Discord BotおよびローカルCLIをインターフェースとして自律動作するハイブリッドな構成を採用しています。

```
+-----------------------------------------------------------------------------------+
|  【開発・指示環境 (人間 / Mac / GPDホスト)】                                         |
|    - SSHターミナル経由での操作                                                       |
|    - Discordクライアント（スマートフォンやブラウザなど、外出先からの操作）             |
+-----------------------------------------------------------------------------------+
                                         │ 指示・メンション
                                         ▼
+-----------------------------------------------------------------------------------+
|  【実行環境 (GPD実機コンテナ: ai_agent_core)】                                      |
|    - discord_agent.py (Gemini 3.1 Flash Lite を使用した自律ツール実行ループ)          |
|    - agent.py (コンテナ内で直接叩く自律開発用 CLI REPL)                             |
|    - ツール群 (ファイル読込、置換、Grep検索、コマンド実行要求)                         |
|    - ホストの /var/run/docker.sock を共有し、コンテナ内からホストDockerを操作          |
+-----------------------------------------------------------------------------------+
                                         │
                                         ▼
+-----------------------------------------------------------------------------------+
|  【安全ゲート (ASEP & 監視システム)】                                               |
|    - コマンド実行前には必ず [✅ 承認] [🛑 却下] ボタンをDiscord等に送信し、             |
|      人間がクリックしない限りシェルコマンドは一切実行されない (ASEP)                 |
|    - ホスト側の独立した monitor.py がヘルスチェックを常時監視。                         |
|      エージェントによるコード改変でコンテナが壊れた場合、直前の正常版 (LKG) へロールバック|
+-----------------------------------------------------------------------------------+
```

### 🔒 セキュリティガードレール
エージェントが利用するファイル操作ツール（`read_file`, `write_to_file`等）には、プログラムレベルで以下のブラックリストが含まれており、プロンプトインジェクション等による機密情報流出やシステムの破壊を防いでいます。
* **ブロック対象**: `secrets/`（APIキー）、`controller/`（監視・ロールバックプログラム）、`.git/`（Git管理領域）、`.env`（環境変数ファイル）

## 🚦 3. 現在の稼働ステータス (Current State)

* **実機検証（L3フェーズ）完全合格**:
  * Discord上で「`make status` を実行して」とメンションすると、エージェントが自動的に実行計画（PLAN）を起票。
  * Discordチャットに **`[✅ 承認 (YES)]` `[🛑 却下 (NO)]`** のボタンを自動添付して返信。
  * 人間が `[承認]` をクリックすると、非同期でコンテナ内からホスト上のコマンドが走り、コンテナやポートのバインド稼働状況（STDOUT）が自動でチャットに返却される一連のループが100%正常動作しています。
* **ローカルCLI REPL (`agent.py`) の自律エージェント化**:
  * コンテナ内で `python3 src/agent.py` を実行するだけで、コンテナに入りながらCLI上で同一の自律開発ツールを使用できます。

## 🗺️ 4. 開発ロードマップ (6 Sprints)

Kanonが自律的に学習し仕事を作る「自己拡張ループ」に向けて、以下の6段階のスプリントで開発を推進します。

1. **Sprint 1 (実行基盤 - 現在進行中)**: **GitHub連携**
   * エージェント専用のGitHub Botアカウントを使用し、「作業用ブランチの作成 ➔ コード修正・テスト ➔ コミット・プッシュ ➔ GitHub APIによるPR（プルリクエスト）の自動起票」までを自律完結させる基盤構築。
2. **Sprint 2 (仕事化)**: **Task Manager**
   * AIが「今やる仕事（タスク）」を管理・優先順位付けするタスク管理機能。
3. **Sprint 3 (監視・観測)**: **Observer**
   * Discord、ログ、Memory、pytest結果などをバックグラウンドで常時読み取る仕組み。
4. **Sprint 4 (プランニング)**: **Planner**
   * `Inbox`（対話ログ、メモ、PDFなどの流入ゲートウェイ）を自動解析し、自ら「Issue（未来の仕事）」を自動起票する仕組み。
5. **Sprint 5 (実作業)**: **Executor**
   * サブエージェントや他のモデル（Gemini/Claude等）にタスクを切り分けて実行を依頼する機能。
6. **Sprint 6 (学習・自己改変)**: **Reflection**
   * タスク完了時の振り返りを `Memory` へ蓄積し、教育の第一層（憲法・設計）を自ら洗練する学習機能。

## 🚀 5. 次回行うタスク (Sprint 1: 自動PR起票フロー)

### 1. 人間によるBot用GitHubアカウントの準備
* 手元のMacからBot専用のGitHubアカウントを作成し、Personal Access Token (PAT) を発行（必要な権限: `repo`）。
* 実機の `ai-agent/secrets/.env` に `GITHUB_TOKEN=ghp_xxxx` としてトークンを記述。

### 2. エージェントによるPR自動作成モジュールの実装
* Python の `urllib.request` または `requests` を使用して、GitHub REST API (`POST /repos/{owner}/{repo}/pulls`) を直接叩いて PR を起票する軽量なPR作成機能を `agent_tools.py` などに実装。
* コード修正の成功時に自動でブランチを切り、プッシュして PR を作成するツール（または一連の自動フロー）を `agent_tools.py` や `agent.py` に組み込む。
