## 1. 🧠 人間の認知モデルとエージェントマッピング

本設計は、Alan Baddeley のワーキングメモリモデル、および Endel Tulving の長期記憶分類モデルをエージェントのシステム構成にマッピングし、「人間の認知制約に準拠した強靭かつ効率的な記憶システム」の構築を目指します。

```
                     [中央実行系 (LLM Planner)]
                                 │
     ┌───────────────────────────┼───────────────────────────┐
     ▼                           ▼                           ▼
[音韻ループ (Chat Context)] [エピソードバッファ (L1: WM)] [視空間スケッチパッド (Workspace)]
                                 │
                     ┌───────────┴───────────┐
                     ▼                       ▼
           [エピソード記憶 (L3)]       [意味記憶 (L4)]
            ・意思決定 (DEC)            ・経歴プロファイル
            ・教訓 (Lessons)            ・プロジェクトコンテキスト
```

### ① Baddeley のワーキングメモリモデルの適用
* **中央実行系 (Central Executive)**: 
  - エージェントにおける LLM の推論・制御部。推論リソースの配分、アテンションの制御を行う。
* **音韻ループ (Phonological Loop)**: 
  - 直前のチャットコンテキスト（直近 1〜2 件の発話テキスト）。LLM が一時的に保持する音声・文字の短期保持。
* **視空間スケッチパッド (Visuospatial Sketchpad)**: 
  - エージェントが現在開いているソースコード、編集中のファイルの空間的配置、リポジトリツリーの構造。
* **エピソードバッファ (Episodic Buffer)**: 
  - 異なる情報源（音韻、視空間、長期記憶）を統合し、時間的な「エピソード（文脈）」として一時保持する場所。これが後述の **L1: Working Memory** に相当する。

### ② Tulving の長期記憶モデルの適用
* **エピソード記憶 (Episodic Memory)**:
  - 時間的・空間的コンテキスト（「いつ・どこで」）を伴う個人の出来事の記憶。➔ **L3 長期エピソード記憶**（決定履歴・Lessons）にマッピング。
* **意味記憶 (Semantic Memory)**:
  - 時間的文脈を持たない一般的な知識や事実の記憶。➔ **L4 長期意味記憶**（人・プロジェクト・規則）にマッピング。
* **手続き記憶 (Procedural Memory)**:
  - 体で覚えるスキルや習慣（自動化された行動）。➔ エージェントにおける Python スクリプト、Git操作などの **自動ツール (Tools)** にマッピング。

## 2. 🗂️ Memory の L0〜L4 階層構造 (Memory Hierarchy)

記憶を生存および処理特性に応じて 5 つの階層に分離し、メモリへの過負荷を防ぎます。

| 階層 | 名称 | 保持期間 | 格納実体 | 特徴と容量制限 |
| :--- | :--- | :--- | :--- | :--- |
| **L0** | Sensory Register (感覚記憶) | 揮発性 (1ステップ) | LLM Context Window | 現在のプロンプト、直前のツール実行結果。 |
| **L1** | Working Memory (ワーキングメモリ) | セッション中維持 | `working_memory.json` | 現在処理中の**アクティブトピック（3〜10件制限）**。 |
| **L2** | Mid-term Memory (中期記憶) | 一時的 (バッチ実行まで) | `memory/inbox/` | 未整理の対話ログ、一時書き出しファイル。 |
| **L3** | Long-term Episodic (長期エピソード) | 永続的 | `memory/decision_history/` | 時間軸と文脈を伴う決定（DEC）や教訓（Lessons）の MD。 |
| **L4** | Long-term Semantic (長期意味記憶) | 永続的 | `memory/{career, people, ...}`| プロファイル、プロジェクト仕様などの構造化定義 MD。 |

### 🧠 Working Memory 容量制限の明示 (マジカルナンバー)
人間の短期記憶能力の限界（マジカルナンバー $7 \pm 2$ または $4 \pm 1$）に準拠し、L1 Working Memory 内のアクティブトピック（スロット）は **最大 3〜10 個** に厳格に制限します。
- スロットが満杯になった場合、最も古いトピック、または優先度の低いトピックは強制的に要約され、L2 (Inbox) もしくは L3 (Episodic) にフラッシュ（書き出し）されて Working Memory から削除されます。
- これにより、LLM の迷子（プロンプトの肥大化による注意散漫）を物理的に防止します。

## 3. 🚌 Read Bus (優先度付きフォールバックチェーン)

エージェントが意思決定や回答生成を行う際、必要な情報へアクセスする**優先経路（Read Bus Priority）**を以下のように定義し、無駄な全件検索や API コストの発生を防ぎます。

```
 [情報要求]
    │
    ├─► 1. L1 (Working Memory) を検索 ➔ [発見] ➔ 終了
    │      (現在アクティブなトピックに存在するか？)
    │
    ├─► 2. L3 (Long-term Episodic) を検索 ➔ [発見] ➔ 終了
    │      (過去に類似するイベントや決定、教訓があるか？)
    │
    ├─► 3. L4 (Long-term Semantic) を検索 ➔ [発見] ➔ 終了
    │      (人物プロファイル、プロジェクト仕様に定義があるか？)
    │
    └─► 4. L2 (Mid-term Memory) を検索 ➔ [発見] ➔ 終了
           (未整理の一時バッファにあるか？)
```

この **priority fallback chain** により、エージェントは常に最も注意を向けるべき「現在アクティブな文脈」から順に情報をドリルダウンし、文脈と合致した最も具体的な意思決定を優先的に採用します。

## 4. 🗂️ Knowledge Taxonomy と日本十進分類法 (NDC) の評価

長期記憶 (L3 / L4) の分類（Taxonomy）設計にあたり、**日本十進分類法 (NDC)** をモデルとして評価します。

### 日本十進分類法 (NDC) の評価
* **評価**:
  - *メリット*: 知識全体を 0〜9 の 10 区分で階層的に表現（十進分類）するため、分類構造が直感的で、ドリルダウンが容易。
  - *デメリット*: 人文科学、歴史、文学などの人間用の分類が多用されており、AI エージェントの生存・ソフトウェア開発を中心としたコンテキストにおいては不要なノイズが多く不適合。
* **結論**:
  NDC をそのまま使用することは見送りますが、その「十進木構造」の設計思想を参考にし、エージェントの認知・生存・開発に最適化した独自の十進タクソノミ **Agent Decimal Classification (ADC)** を構築候補として評価します。

#### 💡 ADC (Agent Decimal Classification) ドラフト
- **000**: Core Rules & Constitutional (エージェント憲章、基礎ルール)
- **100**: Decision History (意思決定ログ、ADR)
- **200**: Project Context (プロジェクト目標、開発進捗、リポジトリ境界)
- **300**: User Profiles / People (ユーザーの好み、関係者の特徴)
- **400**: Agent Career & History (エージェント自身のスキル、更新履歴)
- **500**: Lessons Learned (教訓、バグ対処パターン、ベストプラクティス)
- **600-900**: *未定義 (将来の拡張用)*

## 5. 境界分離: Knowledge Model は後段フェーズへ

* **本フェーズ (Phase 2) の境界**:
  本フェーズでは、「L0〜L4 階層の格納」、「Working Memory (3〜10個) のスロット制御」、「優先度付き Read Bus の実装」という**メモリの入出力と階層管理のインフラ構築**に専念します。
* **後段フェーズ (Phase 3 以降) への分離**:
  意味的な関係性をグラフ化する「ナレッジグラフ（知識グラフ）」や「概念オントロジー（Knowledge Model）」の構築、動的な知識の推論エンジンは、長期記憶の永続化インフラが安定した後の **Phase 3 以降**に完全に分離します。
