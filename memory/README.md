# 長期記憶保管庫 (Memory Architecture)

本ディレクトリは、`kanon` が保持する長期記憶（事実、経験、人物情報、判断履歴、環境のヘルス状態等）を蓄積する一次ソース領域です。

---

## 🎨 1. メモリ運用の基本方針

* **データ形式**:
  すべての情報は人間の可読性とエージェントの処理効率を両立するため、**Markdown（`.md`）形式**で記録します。
* **履歴・監査性の維持**:
  情報の追加、修正、削除はすべて **Git コミット履歴**によってバージョン管理され、時間軸に沿った変化を差分として監査・追跡できるようにします。
* **構造化メタデータ (Frontmatter)**:
  各ファイルの上部には、必ず以下の YAML フロントマッターを記述し、メタデータを明示してください。
  ```yaml
  ---
  category: (e.g., career, health, decision)
  last_updated: YYYY-MM-DDTHH:MM:SS±HH:MM
  importance: (1 to 5, where 5 is critical)
  summary: (Brief summary of the memory entry)
  ---
  ```

## 📂 2. サブディレクトリの責務定義

### ① `inbox/` (未整理データ・一時メモ)
* **責務**: エージェントが外部から取得した一時的なテキスト、要約待ちのウェブページ、または人間からの即時メモの一時退避場所。
* **追加予定要素**: 定期的なエージェントタスクにより、整理された後、適切な他のメモリディレクトリへ振り分けられ、inboxからは削除されます。

### ② `projects/` (プロジェクト状態管理)
* **責務**: 人間（ユーザー）およびエージェント自身が関わっている、または過去に関わったプロジェクトの詳細、進捗、直近の目標、リソース情報の蓄積。
* **追加予定要素**: 各プロジェクトの「ビジョン」「直近のマイルストーン」「重要な成果物へのリンク」。

### ③ `career/` (経歴・実績アーカイブ)
* **責威**: 人間のキャリア、経歴、学歴、職歴、保有スキル、および執筆物・開発物といった成果物の記録。デジタルツインが人間の代理人として振る舞う際のバックグラウンド知識となります。
* **追加予定要素**: スキルマップの評価（定期更新）、業務履歴シート、過去の代表的ポートフォリオのインデックス。

### ④ `health/` (健康・リソースログ)
* **責務**: ホストPCのリソース状態（CPU、メモリ、ディスク使用量）、コンテナ状態、エージェント自身の健康スコア、および（必要に応じて）ユーザー自身の活動記録やライフログ。
* **追加予定要素**: リソースの月次・週次サマリーレポート、コンテナクラッシュログの履歴。

### ⑤ `people/` (関係性・人物マスター)
* **責務**: 家族、友人、同僚、ビジネス関係者などの連絡先、関係性、過去の対話や約束事のサマリー。
* **追加予定要素**: 対話タイムライン（誰と・いつ・何を話し、どのような約束をしたか）、ギフトやイベントの履歴。

### ⑥ `decision_history/` (人生およびシステム意思決定ログ)
* **責務**: 技術的な決定（ADR）とは別に、人間としての重要な意思決定（就職、購買、人生の決断）やエージェントの重要な自律判断の履歴と、その判断に至った背景（Why）の記録。
* **追加予定要素**: 選択肢（Pros/Cons）、決定理由、決定時期、関わった人物。
